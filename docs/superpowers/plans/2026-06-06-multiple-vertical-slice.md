# Plan: MULTIPLES — toolchain → engine → Tier-1 vertical slice

## Goal
Ship a runnable Tier-1 vertical slice of MULTIPLES on Android (dev on Windows): a pure-Dart deterministic rules engine proven by tests, wrapped in a minimal Flutter UI that plays one tier including the signature multiple-arbitrage merge, with local save/resume and a balance sim harness.

## Architecture
Monorepo, two Dart packages (clean dependency arrow points one way: app → engine, never the reverse):

```
multiple/                       (git root)
├─ packages/engine/             PURE DART. No flutter import. Runs under `dart test`.
│  ├─ lib/
│  │  ├─ money.dart             fixed-point helpers (cents, milli, bp) + format
│  │  ├─ rng.dart              SplitMix64 stream + explicit cursor
│  │  ├─ model.dart            Venture, GameState, Sector, Action union
│  │  ├─ resolver.dart         pure apply(state, action) -> state; net worth getter
│  │  ├─ content.dart          typed Card/Economy loaded from JSON
│  │  └─ engine.dart           barrel export
│  ├─ assets/                   cards.json, economy-model.json (copied from /data)
│  └─ test/                     red-green-refactor lives here
├─ app/                         Flutter app. Depends on engine via path.
│  └─ lib/                      screens, widgets (grey-box first, per docs/05)
├─ data/  docs/  prototype/     (existing — design source of truth)
└─ .github/workflows/ci.yml     dart test on every push
```

**Single source of truth for fixed-point (locked, do not deviate):** money = integer **cents**; `multipleMilli` = milli-units **×1000** (14x = `14000`); `ownershipBp` = basis points **×10000** (80% = `8000`). Truncate toward zero on every multiply/divide. No `double` anywhere in `packages/engine`.

**The five mutable inputs (the §7 invariant):** `{ebitda, multiple, netDebt, own, cash}`. `ebitda/multiple/netDebt/own` are **per-venture**; `cash` is the single global. Net worth is a **derived getter, never stored**.

## Tech Stack
- Dart SDK 3.x (via Flutter SDK, stable channel), Flutter 3.x.
- `package:test` for the engine; `flutter_test` for the app.
- `json_serializable` + `build_runner` for content codegen (CI step).
- Android toolchain (Android Studio + SDK) — **deferred to Phase 3**; Phases 1–2 need only the Dart SDK.
- iOS — deferred entirely (Codemagic cloud-Mac CI later, per `data/stack-decision.json`).

## Conventions
- Conventional commits. Branch off `main`; do not commit to `main` directly (per global rules) — work on `feat/*` branches, open a PR per phase.
- Every engine task is red → green → refactor. Run `dart test` (cwd `packages/engine`).
- Expected outputs are written into each task; if reality differs, stop and reconcile against the docs, do not patch the test to pass.

**Commit:** `feat(engine): deterministic SplitMix64 RNG with explicit cursor`

### Task 1.5: `apply(state, action)` per-action branches
**Files:** `lib/resolver.dart` (extend), `test/action_*_test.dart` (one file per action)
**Red→Green→Commit per action** (11 small tasks, 2–5 min each), each asserting pre/post-conditions from docs/02:

**Commit each:** `feat(engine): resolve <Action> with pre/post-conditions`

### Task 1.6: The §7 invariant test (most important test in the project)
**Files:** `test/invariant_test.dart`
**Red/Green:** load `assets/cards.json`; assert every card's `deltas` keys ⊆ `{ebitda,multiple,netDebt,own,cash}`; assert `apply` for every context-free action only mutates those fields + bookkeeping (diff GameState shape, exclude `rngCursor/actionLog/round/tier`); assert no public setter writes net worth.
**Run:** `dart test test/invariant_test.dart` → Expected: PASSED, 33 cards checked, 0 violations.
**Commit:** `test(engine): enforce §7 mutate-only-five-inputs invariant`

### Task 1.7: Golden draw-order contract
**Files:** `test/golden_draworder_test.dart`, `test/golden/draworder_seed42.txt`
**Red/Green:** run a fixed scripted sequence of rounds from seed 42; snapshot the exact draw order + RNG cursor to a golden file; test fails if the sequence diverges (guards replay determinism that the AUTOPSY screen + sim harness depend on).
**Run:** `dart test test/golden_draworder_test.dart` → PASSED.
**Commit:** `test(engine): pin golden draw-order for replay determinism`
